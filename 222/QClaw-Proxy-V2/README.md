# QClaw Proxy V2 - 专业多账户管理版

## 项目简介
QClaw Proxy V2 是在 V1 纯净版基础上进行的重大升级。它不再仅仅是一个简单的转发脚本，而是一个完整的**本地大模型资源网关**。

通过 V2 版本，你可以轻松实现：
*   **多账号收割**：一键将多个 QClaw 账号凭证存入本地池。
*   **智能轮询 (Round-Robin)**：自动在多个账号之间分配请求负载。
*   **自动故障切换**：单账号限流 (429) 或失效 (401) 时自动尝试下一个账号。
*   **可视化监控**：通过高颜值 Web 面板实时观察每个账号的运行状态。

为了避免与 V1 版本冲突，V2 采用了全新的独立端口设计：

*   **API 代理服务 (`18832`)**：
    *   接口地址：`http://localhost:18832`
    *   用途：填入 Claude Code / CC Switch 的 API Base 设置。
*   **管理后台 UI (`18833`)**：
    *   访问地址：`http://localhost:18833`
    *   用途：收割账号、管理凭证池、查看请求统计。

## 🚀 快速开始

### 1. 启动服务
双击运行目录下的 **`启动QClaw管理版.bat`**。
*   程序会自动安装所需依赖。
*   启动成功后，你会看到两个服务的运行地址。

### 2. 收割多个账号（建立令牌池）
1.  在浏览器访问 `http://localhost:18833` 进入管理面板。
2.  打开 **QClaw 官方客户端**，登录你的第一个账号。
3.  回到管理面板，点击右上角的 **“同步当前账号”**。
4.  在 QClaw 客户端退出并登录你的第二个账号。
5.  再次回到管理面板，点击 **“同步当前账号”**。
6.  重复此过程，直到你所有的账号都出现在卡片列表中。

### 3. 配置客户端
以 Claude Code 为例，在设置中修改：
*   **API Base**: `http://localhost:18832`
*   **API Key**: 任意填写 (如 `sk-multi`)
*   **Model**: `claude-3-5-sonnet-20241022`

## 💡 功能亮点

### 1. 深度思考标签清洗
内置了针对推理模型（如 DeepSeek R1）的 `<think>` 标签实时剥离逻辑，确保 Claude Code 界面输出纯净的代码或回答。

### 2. 自动负载均衡
当你进行高频操作时，代理会自动按顺序切换账号，极大降低了触发官方频率限制（Rate Limit）的概率。

### 3. 玻璃拟态管理界面
实时展示每个账号的成功/失败次数，让你对自己的“资源池”了如指掌。

## 常见问题排查
*   **端口占用**：如果 18832 或 18833 被其他程序占用，请在 `app.py` 末尾修改端口号。
*   **收割失败**：请确保 QClaw 客户端处于已登录状态且版本较新。
*   **依赖问题**：若 `.bat` 运行报错，请手动执行 `pip install flask requests pywin32 cryptography`。
